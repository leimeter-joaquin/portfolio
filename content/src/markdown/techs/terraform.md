# Where I Am with Terraform

Terraform is something I'm actively learning. My background is primarily front-end and full-stack JavaScript/TypeScript, but I've become increasingly interested in the infrastructure layer — how applications are deployed, what they run on, and how that infrastructure is managed over time.

Terraform's declarative model appeals to me for the same reasons typed languages do: you describe the desired state, and the tool figures out how to get there. It makes infrastructure reviewable, version-controlled, and reproducible.

## What I Am Exploring

- Core HCL syntax: resources, variables, outputs, and data sources
- State management and how Terraform tracks what it has provisioned
- AWS as a provider: basic compute, storage, and networking resources
- Modules for reusable infrastructure patterns
- How Terraform fits into a CI/CD workflow

## Context

This is early-stage learning — I wouldn't yet claim production experience with Terraform. I'm including it here because it is a deliberate part of how I am growing as an engineer, and I expect it to become a meaningful part of my work over the next year.
