# Why CSS Matters to Me

CSS is one of the places where I feel most confident. I enjoy the craft of it — getting layouts right, building consistent design systems, and writing styles that scale without becoming a mess. Most of my front-end work has involved owning the visual layer alongside the logic, so I've had to develop real depth here.

I've worked with plain CSS, SCSS, Tailwind, utility class generators like Gorko, and scoped styles in Vue and Astro. I'm comfortable picking the right approach for the project rather than defaulting to whatever I know best.

## What I Focus On

- Layout systems: Flexbox and Grid used deliberately, not interchangeably
- Design tokens: centralising colors, spacing, and scale so the system is consistent and easy to change
- SCSS: variables, maps, mixins, and `@use`/`@forward` over `@import`
- Utility class systems: generating utilities from a config rather than writing one-off classes
- Scoped styles in component frameworks (Vue SFC, Astro) — keeping specificity low and styles predictable
- Accessibility: semantic HTML paired with styles that support screen readers and keyboard navigation

## Real-World Usage

I've worked with CSS and SCSS across all my front-end projects. I built this portfolio's design system using Gorko as a utility generator, with SCSS for component-level styles. At Brookfield I worked with Tailwind at scale across a multi-package monorepo. At Paytient, accessibility requirements pushed me to be more deliberate about how styles and semantics work together.
