# React in My Work

React is not my primary framework, but I've shipped real production work with it. At Paytient I worked on a React and Next.js codebase for over a year, building and maintaining front-end features for a healthcare payments product.

Coming from a Vue background, I had to adapt — the mental model is different, particularly around state, effects, and how components compose. That context switch sharpened my understanding of both frameworks: I can see clearly what each one optimises for and where the trade-offs are.

## What I Focus On

- Hooks: `useState`, `useEffect`, `useContext`, and custom hooks for shared logic
- Component composition and avoiding unnecessary re-renders
- Next.js routing, data fetching patterns, and the distinction between server and client components
- Accessibility-first component design: semantic HTML, ARIA where needed, keyboard interaction
- Writing components that are easy to test and reason about

## Real-World Usage

My React experience comes from Paytient (Apr 2023 – Sep 2024), where I worked on a healthcare wallet product with a user base that included older adults. Accessibility was a real requirement, not an afterthought, which pushed me to build components more carefully than I might have otherwise.
