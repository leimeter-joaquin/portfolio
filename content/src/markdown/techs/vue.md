# Why I Use Vue.js

Vue is where most of my front-end experience lives. I've worked with it across teams of different sizes and project types — from a large enterprise codebase with a multi-package monorepo to smaller, fast-moving internal tools. I've shipped real features in both Vue 2 and Vue 3, and I've been through the migration between them.

It fits the way I think about UI: composable, readable, and predictable. The single-file component structure keeps logic, template, and styles together without feeling cluttered, and the Composition API makes it easier to reason about complex state and side effects.

## What I Focus On

- Composition API and composables for sharing logic cleanly across components
- State management with Pinia — including migrating Vuex stores to Pinia
- Reactive patterns and how Vue's reactivity system behaves under the hood
- Component design: props, emits, slots, and keeping components focused
- Working within multi-package Vue monorepos and understanding build boundaries

## Real-World Usage

I've used Vue as my primary tool at Brookfield Residential, Radium Rocket, and AlixPartners. At Brookfield I built a complex URL-driven filtering system and contributed to the Vue 2 → Vue 3 migration. At AlixPartners I use Vue daily building internal platform tools for consultants.
