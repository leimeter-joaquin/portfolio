# SQL and Databases in My Work

SQL has come up in a few different contexts throughout my career. At PwC I wrote queries to extract and shape data for reporting dashboards. At Radium Rocket I worked alongside a SQL engineer on a database consolidation effort — deduplicating records, writing normalization scripts, and migrating data from two separate schemas into one.

PostgreSQL specifically I've used as the database layer in a Node/Express back end at Radium Rocket. That experience was more about understanding the data model and working within an existing schema than designing one from scratch.

## What I Am Exploring

- Query writing: joins, aggregations, subqueries, and window functions
- Schema design and normalization
- PostgreSQL-specific features: indexes, constraints, and query planning basics
- Using SQL alongside an ORM vs writing raw queries and when each makes sense
- Data migration and deduplication patterns

## Context

My SQL experience is real but not deep — I've used it as a supporting skill rather than as a specialisation. I'm including it because it has been practically relevant across multiple projects and I'm actively building on that foundation.
